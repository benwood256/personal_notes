** Booting applications

I can save files (in a specific way) in ~/.config/autostart/ so they are opened on boot

The standard format is (below example is for Emacs):

[Desktop Entry]
Type=Application
Name=Emacs
Exec=emacs
Terminal=false
X-GNOME-Autostart-enabled=true

*** ~[Desktop Entry]~

- Starts a desktop entry file
- Required header

*** Type=Application

- Says this launches a program
- Other types do exist (Link, Directory, etc...)

*** Name=Emacs

- The human-readable name
- This is what appears in startup app lists (i.e. Startup Application I used to have)

*** Exec=emacs

- The command GNOME runs after login
- It must be a command that works from the terminal

*** Terminal=false

- Do not open a terminal window for it
- GUI apps should normally use this

*** X-GNOME-Autostart-enabled=true

- Tells GNOME to run it
- If set to false, it is ignored

** Installation

*** Download Steps

1. .iso file
    You need to download a .iso file specific to Arch (from the Arch wiki)
    This is a single file that contains everything on a CD -> file system & folder structure
    The Arch iso file contains enough to boot itself & then complete installing the OS system

2. A tool to write an ISO to a USB so it’s bootable
    The computer can’t boot from a file as the file is sitting in my Fedora environment, so on boot the computer can’t get access to this file as it needs to boot Fedora first
    At boot, the UEFI looks for bootable disks (i.e. USB or SSD) or EFI bootloaders -> it doesn’t look for .iso files
    Therefore, the ISO file needs to be placed on a USB, so then the UEFI can read the USB
    Again though, the UEFI can’t read a file on a USB, the USB needs to be converted into a copy of a bootable disk
    So, a tool will extract the data from the .iso file & convert it away from using files, it then writes this data to a USB & now the USB can act as a bootable disk drive

3. Boot & select Arch

*** Setting up the OS

1. Connect to wifi

2. Partition a disk
    You need 3 partitions
    EFI System partition -> reuse the one I created when I installed Fedora
    Linux root partition -> creates the Arch file system 
    Swap partition (optional) -> Linux uses as backup memory (RAM extension) when physical RAM is full

3. Format the root partitions
    Formatting a partition means to create a file system on it
    So, the OS can know how to organise, store & retrieve files
    Technically, partitioning a disk means to allocate an empty space of data for something, at this stage, it’s just an empty space allocated to a certain size
    Then formatting, tells it to install x filesystem (i.e. the root)

4. Encrypt the root partition

5. Install LVM (Logical Volume Manager)
    This is a software preinstalled on Linux
    LVM will take a partition & break up the data into logical units, then if a certain unit needed more data, you can shrink the size of 1 to increase the size of another (move data around)
    Often, in an Arch Linux system a root partition is made & a /home partition is made (/home is for personal files which sit in an entirely separate partition to system files (root)), this way, you can reinstall the OS without deleting personal files
    LVM will create a shared pool of data for these 2 partitions, which, if down the line the /home directory uses all it’s data but the root still has a lot spare, you can transfer over data (there are other uses for LVM other than / & /home but this is the best beginner example)
